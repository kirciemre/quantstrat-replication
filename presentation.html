02 / 10Paper Overview

What We're Replicating

Section 4: Synthetic experiments on regime-switching mean-reverting signals

The Trading Environment

Signal St follows a regime-switching OU process where mean θ, speed κ, and volatility σ are latent Markov chains invisible to the agent.

St+1 = St + κt(θt − St)Δt + σt√Δt · Z

The Reward Function

Position P&L minus transaction cost λ. Agent controls inventory It ∈ [−10, 10].

rt = It+1(St+1 − St) − λ|It+1 − It|

Three Complexity Scenarios

  • Scenario 1: θt switches (κ, σ fixed)
  • Scenario 2: θt and κt both switch
  • Scenario 3: θt, κt, σt all switch

Markov Chain Transitions

Regime transitions follow continuous-time Markov chains. Transition matrix computed via matrix exponential.

P(Δt) = expm(A · Δt)   Δt = 0.2

Expected Ordering (Table 4)

prob-DDPG > hid-DDPG > reg-DDPG — Bayesian posterior is the richest information state, giving the highest expected reward under partial observability.

Regime-switching OU simulation, faithful to paper parameters

RegimeProcess — Markov Chain

class RegimeProcess: # Transition matrix via matrix exponential P = expm(A * dt) # Eq. (5) cumP = cumsum(P, axis=1) def sample_path(batch, length): # Initial regime from stationary distribution idx[:,0] = choice(n, p=stationary) # Forward steps via inverse CDF idx[:,i] = argmax(u < cumP[idx[:,i-1]])

Euler-Maruyama Discretisation

Si+1 = Si + κi(θi − Si)Δt + σi√Δt · Zi

Δt=0.2 (Table 1). Initial S₀ ~ N(μ_inv, 3σ_inv) where σ_inv = σ/(2κ) per paper's literal formula (not textbook √(2κ)).

06 / 10Neural Network Layer

autodiff.py & nn.py

Micrograd-style autograd engine + GRU from scratch — zero ML dependencies

Custom Autograd — Why?

PyTorch/JAX unavailable. Implemented Karpathy micrograd-style: each operation creates a graph node and stores its ∂-backward closure. Supports batch matmul with broadcasting.

  • Ops: +, −, ×, ÷, matmul, tanh, softmax, SiLU, log, LeakyReLU
  • Topological sort → reverse-mode accumulation
  • _unbroadcast(grad, shape) handles batch dims

GRU — Paper's Literal Eq. (6-9)

p = tanh(x @ Hp + h_prev @ Up + bp) z = tanh(x @ Hz + h_prev @ Uz + bz) c = tanh(x @ Hh + (p*h_prev) @ Uh + bh) h = (1-z)*h_prev + z*c

⚠ All 3 gates use tanh (paper literal). Standard PyTorch GRU uses sigmoid for reset/update — a key fidelity difference.

Two-Mode Inference Strategy

Frozen GRU/head networks evaluated in pure NumPy during DDPG training — no graph construction, ~10× faster.

# Reads .data directly, no autodiff graph def _gru_numpy(gru_stack, S_window): for cell in gru_stack.cells: p = np.tanh(x @ cell.Hp.data + ...) # hid-DDPG online update uses Tensor

Numerical Stability (3 Layers)

  • Glorot init in all Linear layers — prevents exploding activations
  • Clip SiLU/sigmoid input to ±30 before exp() — prevents overflow
  • Grad norm clipping (max_norm=5) in pre-training — prevents MSE = 10²⁴ on step 1
07 / 10DDPG Training

ddpg_core.py & agents.py

Algorithm 1 — one agent-agnostic train_iteration() drives all three agents

Training Loop — 4 Steps Per Iteration m

1

hid_gru_update()

hid-DDPG only: MSE on next-step prediction → backprop GRU → step opt_gru. No-op for prob/reg.

2

Critic update (ℓ=1)

y = r + γ·Qtgt(Gt+1, π(Gt+1)). MSE loss. Current actor, target critic — Eq. (12).

3

Actor update (l=5 steps)

Maximise Q(Gt, π(Gt)). Loss = −mean(Q). Gradients update actor only.

4

Polyak target update

θtgt ← (1−τ)·θtgt + τ·θ   where τ=0.001

build_state() Interface

Agent State Gt dim
prob [S̄, Ī, π̂₀, π̂₁, π̂₂] 5
hid [S̄, Ī, (h+1)/2] 12
reg [S̄, Ī, S̃̄t+1] 3

Exploration Noise Schedule

ε(m) = max(a / (a + m), ε_min)   a = 100

Decays from ~1.0 → ~0.01 over 10k steps. Gaussian noise ε·N(0,1) added to actor output, clipped to [I_min, I_max].

StatefulPolicy (Eval Wrapper)

Roller's API only passes signal window, not It. StatefulPolicy tracks inventory internally, staying in sync with the roller's own I update at every step.

08 / 10Paper Ambiguity Challenges

Challenges: Paper Ambiguities

Four underspecified details that broke v1's prob-DDPG ordering

❌ GRU Gate Activation

Paper Eq.(6-9) says "σ is the hyperbolic tangent" for all three gates. Standard PyTorch GRU uses sigmoid for reset/update. v1 used PyTorch default — incorrect per literal reading.

✅ Implemented paper literally

v2 GRUCell: tanh for all gates (reset p, update z, candidate h̃). Documented in config.py note 4.

❌ σ_inv Formula Mismatch

Paper: σ_inv = σ/(2κ). Textbook OU: σ/√(2κ). v1 used textbook — overstretching S₀ initialisation and degrading normalised features at runtime.

✅ Paper literal formula in v2

σ_inv = σ/(2κ) by default. USE_TEXTBOOK_OU_STD flag allows controlled comparison.

❌ Discount Factor γ Too Low

v1 used γ=0.99. Paper Table 8 reports γ=0.999. With γ=0.99 the agent discounts long-horizon mean-reversion gains too aggressively — especially harmful to prob-DDPG which relies on regime persistence.

✅ γ = 0.999 in v2

config.py sets γ=0.999 with explanatory note that Section 4 does not override Table 8's value.

❌ Lookback Window Overrides Missed

Footnote 6: prob-DDPG uses W=20 for Scenario 3 (not W=10). Table 2: reg-DDPG uses W=50 everywhere. v1 used W=10 across the board.

✅ Per-scenario W in config.py

Each SCENARIOS entry stores lookback_prob, lookback_reg, lookback_hid — applied automatically by agents.py at instantiation.

09 / 10Engineering Challenges

Challenges: Engineering & Implementation

Pitfalls specific to building from-scratch RL with a custom autodiff engine

❌ Autodiff Graph Bloat

Calling GRU forward (Tensor ops) inside build_state() during DDPG training builds a new graph every batch. With W=50, B=512, N=10,000: ~280M graph nodes — memory and speed catastrophe.

✅ Dual-mode inference

Training: autodiff only for networks being updated.
Inference: pure NumPy (reads .data) — no graph, ~10× faster. hid-DDPG's online GRU update is the only true autodiff call per iteration.

❌ Inventory Not Passed to Policy

TestEpisodeRoller.run() only provides the signal window to policy_fn. But Gt requires It. A naive wrapper always uses I=0 → completely wrong actions throughout evaluation.

✅ StatefulPolicy wrapper

Maintains self.I internally. Every call: compute action → update self.I = I_next. Stays exactly in sync with the roller's own inventory tracking.

❌ Numerical Overflow in Early Training

Random weight init → large pre-activations → exp(-x) overflows. W=50 regressor hit MSE = 10²⁴ on step 1. SiLU/sigmoid threw RuntimeWarning in both forward and backward passes.

✅ Three-layer stability

1) Glorot init on all Linear weights.
2) Clip input to ±30 in SiLU/sigmoid before exp().
3) Grad norm clipping (max_norm=5) in pre-training loops.

❌ v1 prob-DDPG Reversed Ordering

Final v1 result: hid 41.4 >> prob −2.7. All four paper-ambiguity bugs compounded — wrong γ, wrong σ_inv, wrong GRU gates, and wrong W each independently degraded prob-DDPG.

✅ v2 Early Evidence — Ordering Restored

With just 10 training steps: prob +0.45 vs hid −1.61 vs reg −0.98 (Sc 1). Correct ordering confirmed. Quantitative match requires full N=10,000 run.
